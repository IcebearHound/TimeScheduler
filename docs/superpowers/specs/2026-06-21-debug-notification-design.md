# 调试通知功能 — 设计文档

**日期:** 2026-06-21  
**状态:** 已批准

## 概述

在现有调试面板（`DebugOverlay.tsx`）中添加通知权限状态显示和手动测试通知按钮，方便开发者验证 Browser Notification API 的权限和显示效果。

## 修改范围

- **文件:** `src/components/DebugOverlay.tsx`
- **不涉及:** App.tsx 轮询逻辑、store、新文件

## 设计细节

### 1. 通知权限状态显示

在调试面板头部（折叠/展开状态栏区域），显示当前 Notification API 权限：

| 权限值 | 显示文本 |
|--------|----------|
| `granted` | `通知: ✓` |
| `denied` | `通知: ✗` |
| `default` | `通知: 未设置` |

通过 `Notification.permission` 获取，每 2 秒轮询更新（仅在面板展开时）。

### 2. 测试通知按钮

在调试面板头部添加 **"📢 测试"** 按钮。

点击逻辑：
1. 不支持 Notification → 无操作
2. 权限 `default` → 调用 `Notification.requestPermission()`，权限状态自动更新
3. 权限 `granted` → 发送测试通知：
   - title: `"测试通知"`
   - body: `"这是一条来自时间规划器的测试提醒"`
   - icon: 使用默认
4. 权限 `denied` → 无操作（状态已显示 ✗）

### 3. UI 位置

```
┌─ 调试面板 ─────────────────────────────────────────┐
│ 🔍 调试面板  undo:3 redo:1 ev:42  通知:✓ 📢测试 清 _ │
├─────────────────────────────────────────────────────┤
│ ... 日志列表 ...                                     │
└─────────────────────────────────────────────────────┘
```

折叠状态下只显示精简信息，不显示通知相关内容。
